🔧 One-time setup: connect Firebase

This app now saves everything to the cloud (Firebase Cloud Firestore) instead of the browser, so every device sees the same clinic accounts, checklists, and notifications. It just needs to be pointed at a free Firebase project once.

  1. Go to console.firebase.google.com → Add project (the free Spark plan is enough).
  2. In the new project, open Build → Firestore Database → Create database (start in production mode).
  3. Go to Project settings (gear icon) → scroll to Your apps → click the </> Web icon → register the app (no hosting needed).
  4. Copy the firebaseConfig object it shows you.
  5. Open index.html in a text editor, find const firebaseConfig = {...} near the top of the <script> section, and paste your values in.
  6. In Firestore → Rules, publish rules so the app can read/write its one collection — see the comment above firebaseConfig in the code for a starter rule set.
  7. Save the file and reload this page (and re-publish to GitHub Pages if that's where it's hosted).

B2B Sales Onboarding Manual

Clinic Partner Onboarding Framework — from first contact to Clinic Success handover. Reference this guide, then track live accounts in the Checklist tab.

Purpose

Why this framework exists

The Account Manager must be competent enough to run the full sales presentation and qualify a clinic without medical support in most cases. The Medical Team is a conditional resource, engaged only when a clinic's clinical questions genuinely exceed the Account Manager's own knowledge — never a default step in the funnel.

This guide covers every phase from prospecting through the formal handover to Clinic Success, and applies to Jr. Account Managers, Account Managers, and the Head of Affiliate & Partnerships. Jr. Account Managers and Account Managers carry identical roles and responsibilities across the funnel — the two positions are differentiated by monthly revenue target, not by scope of work.
